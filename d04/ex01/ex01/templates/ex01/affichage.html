Processus d'affichage d'une page statique

Voici le cycle complet de la requête à la réponse pour l'affichage d'une page statique dans Django, à partir d'un simple template.

1. La requête HTTP arrive

L'utilisateur saisit une URL dans son navigateur (ex : /ex01/affichage). Le navigateur envoie une requête HTTP GET au serveur Django.

2. Le routeur URL analyse la requête

Django consulte le fichier urls.py principal du projet, qui redirige vers les urls.py de chaque application grâce à la fonction include(). Django parcourt les patterns d'URL dans l'ordre et s'arrête au premier qui correspond.

3. La vue est appelée

Une fois l'URL reconnue, Django appelle la fonction de vue associée (définie dans views.py). La vue reçoit l'objet HttpRequest en paramètre. Pour une page statique, la vue se contente d'appeler render(request, 'template.html') sans logique supplémentaire.

4. Le moteur de templates compile le template

Django localise le fichier HTML dans les dossiers templates/ des applications listées dans INSTALLED_APPS. Le moteur de templates charge et compile le fichier. Dans le cas d'une page statique, aucune variable de contexte n'est substituée.

5. La réponse HTTP est retournée

La fonction render() retourne un objet HttpResponse contenant le HTML généré avec un code de statut HTTP 200. Django transmet cette réponse au navigateur, qui affiche la page.

Schéma récapitulatif

Navigateur
    │
    │  GET /ex01/affichage
    ▼
Django (urls.py)
    │
    │  Pattern reconnu → appel de la vue
    ▼
views.py → render(request, 'ex01/affichage.html')
    │
    │  Chargement + compilation du template
    ▼
Moteur de templates (Django Template Engine)
    │
    │  HTML généré
    ▼
HttpResponse (status 200)
    │
    ▼
Navigateur affiche la page
