Django, framework web

Qu'est-ce que Django ?

Django est un framework web Python de haut niveau qui favorise un développement rapide et une conception propre et pragmatique. Il a été conçu pour aider les développeurs à créer des applications web de façon rapide, avec moins de code à écrire.

Django suit le principe DRY (Don't Repeat Yourself) : chaque information ne doit exister qu'en un seul endroit. Il intègre nativement de nombreux outils (ORM, système de templates, gestion des formulaires, authentification, admin, etc.) ce qui en fait un framework dit « batteries included ».

Historique

  • 2003 — Django est créé par Adrian Holovaty et Simon Willison au sein du journal Lawrence Journal-World (Kansas, États-Unis) pour gérer des sites d'actualité à fort trafic.
  • Juillet 2005 — Django est publié en open-source sous licence BSD. Le framework est nommé d'après le guitariste de jazz manouche Django Reinhardt.
  • 2008 — Création de la Django Software Foundation (DSF), organisation à but non lucratif chargée de maintenir et promouvoir le projet.
  • 2010 — Django 1.2 introduit le support multi-base de données.
  • 2013 — Django 1.5 ajoute le support partiel de Python 3 ; Django 1.7 (2014) introduit les migrations natives.
  • 2017 — Django 2.0 abandonne le support Python 2 et simplifie la syntaxe des URLs.
  • 2019–aujourd'hui — Les versions 3.x, 4.x et 5.x apportent le support natif de l'asynchrone (ASGI), des améliorations ORM, et une modernisation continue.

Architecture MTV

Django adopte un patron d'architecture MTV (Model – Template – View), proche du MVC classique :

  • Model : définit la structure des données et interagit avec la base via l'ORM.
  • Template : gère la présentation (HTML) rendue à l'utilisateur.
  • View : contient la logique métier, reçoit les requêtes HTTP et retourne des réponses.
